
# API

## Overview

The communication subsystem serves as the bridge between the remote controller HMI and the rover’s local communications system. It establishes and maintains a Bluetooth connection, verifies connectivity, and confirms messages are properly transmitted between systems.

This subsystem:
- Initiates and verifies Bluetooth connection
- Forwards commands from HMI to rover subsystems
- Relays responses back to the HMI

Most messages pass through this subsystem instead of being generated by it.

---

## Messages Sent by Wireless Communication Subsystem

### Bluetooth Connection Check

**Packet Format:**  
`AZcwBT:S:check;YB`

**Byte Length:** 17 bytes

| Field | Value |
|------|------|
| Variable Name | bt_request |
| Variable Type | char[] |
| Min Value | check |
| Max Value | check |
| Example | check |

---

## Messages Received by Wireless Communication Subsystem

### Bluetooth Connection Status

**Packet Format:**  
`AZwcBT:S:T;YB` or `AZwcBT:S:F;YB`

**Byte Length:** 13 bytes

| Field | Value |
|------|------|
| Variable Name | bt_status |
| Variable Type | char |
| Min Value | F |
| Max Value | T |
| Example | T |

---

## Broadcast Messages

### Start Broadcast

**Packet Format:**  
`AZhXST:S:Start;YB`

**Byte Length:** 17 bytes

| Field | Value |
|------|------|
| Variable Name | ST |
| Variable Type | char[] |
| Min Value | Start |
| Max Value | Start |
| Example | Start |

---

### Broadcast Behavior

| Condition | Action |
|----------|--------|
| Message received | All subsystems start and turn on LED |
| Acknowledgment received | System confirms start success |
| No response (~30s) | Report start failure |

---

## Notes
The Wireless Communication Subsystem primarily acts as a relay layer, making sure correct delivery of all messages between the controller and rover subsystems.
