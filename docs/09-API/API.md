# API

## Overview

The Wireless Communication Subsystem serves as the bridge between the remote controller HMI and the rover’s local communications system. It establishes and maintains a WiFi-based MQTT connection, verifies connectivity, and confirms messages are properly transmitted between systems.

On the rover side, the wheel subsystem acts as the local communication point of contact, interfacing with its ESP32 to receive and distribute messages to the rest of the rover subsystems.

This subsystem:

- Establishes and verifies MQTT connection
- Forwards commands from HMI to rover subsystems
- Relays responses back to the HMI

Most messages pass through this subsystem instead of being generated by it.

## Messages Sent by Wireless Communication Subsystem

### Connection Check

**Packet Format:**  
`AZcwBT:S:check;YB`

**Byte Length:** 17 bytes

| Field | Bytes | Section Name | Type | Min Recognized | Max Recognized | Example |
|------|------|--------------|------|----------------|----------------|--------|
| Variable | 2 | BT | string | -- | -- | BT |
| Separator | 1 | : | char | : | : | : |
| Type Identifier | 1 | S | char | S | S | S |
| Separator | 1 | : | char | : | : | : |
| Value | 5 | check | string | check | check | check |
| Terminator | 1 | ; | char | ; | ; | ; |

**Meaning:**

- AZ = Start of message
- c = Communication subsystem
- w = Wheels (Rover local comms subsystem)
- BT = Message type (connection check)
- S = String data type
- check = Request to verify communication link
- YB = End of message

## Messages Received by Wireless Communication Subsystem

### Connection Status

**Packet Format:**  
`AZwcBT:S:T;YB` or `AZwcBT:S:F;YB`

**Byte Length:** 13 bytes

| Field | Bytes | Section Name | Type | Min Recognized | Max Recognized | Example |
|------|------|--------------|------|----------------|----------------|--------|
| Variable | 2 | BT | string | -- | -- | BT |
| Separator | 1 | : | char | : | : | : |
| Type Identifier | 1 | S | char | S | S | S |
| Separator | 1 | : | char | : | : | : |
| Value | 1 | T/F | char | F | T | T |
| Terminator | 1 | ; | char | ; | ; | ; |

**Meaning:**

- AZ = Start of message
- w = Wheels (Rover local comms subsystem)
- c = Communication subsystem
- BT = Message type (connection status)
- S = String data type
- T = Connected (True)
- F = Disconnected (False)
- YB = End of message

## Broadcast Messages

This broadcast message tells every subsystem connected to turn on their LED to indicate the system is ready. It is typically sent from the HMI subsystem and received by all subsystems (destination `X`). The wireless communication subsystem receives and forwards this message across the MQTT network.

### Start Broadcast

**Packet Format:**  
`AZhXST:S:Start;YB`

**Byte Length:** 17 bytes

| Field | Bytes | Section Name | Type | Min Recognized | Max Recognized | Example |
|------|------|--------------|------|----------------|----------------|--------|
| Variable | 2 | ST | string | -- | -- | ST |
| Separator | 1 | : | char | : | : | : |
| Type Identifier | 1 | S | char | S | S | S |
| Separator | 1 | : | char | : | : | : |
| Value | 5 | Start | string | Start | Start | Start |
| Terminator | 1 | ; | char | ; | ; | ; |

**Meaning:**

- AZ = Start of message
- h = HMI subsystem
- X = Broadcast to all subsystems
- ST = Start message type
- S = String data type
- Start = Command to begin system operation
- YB = End of message

---

### Broadcast Behavior

| Condition | Action |
|----------|--------|
| Message received | All subsystems start and turn on LED |
| Acknowledgment received | System confirms start success |
| No response (~30s) | Report start failure |

## Notes

The Wireless Communication Subsystem primarily acts as a relay layer, ensuring correct delivery of all messages between the controller and rover subsystems over the MQTT communication network.
